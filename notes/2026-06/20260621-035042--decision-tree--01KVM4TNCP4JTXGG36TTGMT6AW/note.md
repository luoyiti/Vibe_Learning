# Decision Tree（决策树）

**一句话总结：** 决策树通过递归地选择最优特征和阈值进行二分，贪心地最小化节点不纯度（基尼/熵），将特征空间划分为轴对齐的矩形区域，产出天然可解释的分类模型。

---

## 1) 数学原理

### 1.1 信息熵（Entropy）

$$H(S) = -\sum_{i=1}^{c} p_i \log_2 p_i$$

熵衡量集合的不确定性：纯节点（全同一类）熵 = 0；各类均匀分布时熵最大。二分类最大熵 = 1，三分类最大熵 ≈ 1.585。

### 1.2 信息增益（Information Gain）

$$\text{IG}(S, A) = H(S) - \sum_{v \in \text{vals}(A)} \frac{|S_v|}{|S|} H(S_v)$$

信息增益 = 父节点熵 − 子节点加权熵。ID3 算法选择信息增益最大的特征。

**偏差问题**：信息增益倾向选择取值多的特征（如用样本 ID 可得最大增益但无泛化力）。C4.5 用**增益率（Gain Ratio）** 修正：

$$\text{GR}(S,A) = \frac{\text{IG}(S,A)}{\text{IV}(S,A)}$$

### 1.3 基尼不纯度（Gini Impurity）

$$G(S) = 1 - \sum_{i=1}^{c} p_i^2$$

基尼不纯度 = 从集合中随机取一个样本并随机分配类别时被分错的概率。CART 算法默认使用基尼。与熵相比无需对数运算，速度更快；两者实际效果差异极小。

### 1.4 卡方检验（Chi-square）

$$\chi^2 = \sum_i \frac{(O_i - E_i)^2}{E_i}$$

CHAID 算法使用卡方检验判断分裂的统计显著性。优点：自带显著性门槛防止过拟合；缺点：只处理离散特征。

## 2) 训练过程（CART 算法）

递归自顶向下构建：

1. 若节点所有样本同类 / 达 max_depth / 样本数 < min_samples_split → 返回叶节点
2. 遍历所有特征和阈值，计算基尼增益
3. 选择基尼增益最大的 (feature*, threshold*)
4. 按阈值二分，递归构建左右子树

**停止条件（预剪枝）**：
- `max_depth`：限制树深度
- `min_samples_split`：节点最少分裂样本数
- `min_samples_leaf`：叶节点最少样本数
- `min_impurity_decrease`：最小不纯度增益

---

## 3) 效果评估与剪枝

### 3.1 过拟合

无限制生长的树可 100% 训练准确率（每叶一个样本），但测试准确率在某个深度后下降。训练误差与测试误差的"剪刀差"是过拟合标志。

### 3.2 预剪枝（Pre-pruning）

通过超参数提前停止树的生长：max_depth、min_samples_split 等。简单高效但需调参。

### 3.3 后剪枝（Post-pruning）

**Reduced Error Pruning (REP)**：用验证集自底向上检查——若剪去子树后验证准确率不降则剪。数据驱动，不需预设深度。

**Cost-Complexity Pruning**（CART 默认）：通过代价参数 α 平衡准确率与树复杂度：$R_\alpha(T) = R(T) + \alpha |T|$

### 3.4 评估指标

- **混淆矩阵**：cm[true][pred] 显示分类细节
- **Accuracy**：整体正确率
- **Precision**：TP / (TP + FP)——预测为正的样本中真正为正的比例
- **Recall**：TP / (TP + FN)——真正为正的样本中被正确预测的比例
- **F1**：2 × P × R / (P + R)——精确率和召回率的调和平均
- **K-Fold 交叉验证**：消除单次划分的随机性

## 4) 真实数据实战（Iris 数据集）

Fisher Iris 数据集：150 样本，4 特征（花萼长度/宽度、花瓣长度/宽度），3 类鸢尾花各 50 样本。

**关键发现**：
- Setosa 线性可分（花瓣长度 < 2.5 即可完美分离）
- Petal Length 和 Petal Width 是最重要的特征（加权基尼增益贡献最大）
- 典型测试准确率 > 95%（即使 max_depth = 3）
- Versicolor 与 Virginica 有重叠区域，是分类难点

---

## 5) 常见误区

- **贪心分裂 ≠ 全局最优**：决策树每步只选局部最优分裂，不保证找到全局最优树结构
- **信息增益偏差**：取值多的特征天然获得更高增益；使用增益率（C4.5）或基尼（CART）修正
- **过拟合是默认行为**：不限制的树必然过拟合；必须剪枝或限制深度
- **决策树不稳定**：训练集微小变化可能导致完全不同的树结构——这也是集成方法（Random Forest, XGBoost）更有效的原因
- **特征重要性不可靠**：单棵树的特征重要性偏向高基数特征；集成方法的重要性更稳定
- **轴对齐分裂的局限**：每个节点只在一个特征上切分，无法捕捉斜向决策边界
