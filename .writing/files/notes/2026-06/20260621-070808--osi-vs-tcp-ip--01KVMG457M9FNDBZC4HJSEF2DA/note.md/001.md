# OSI 参考模型与 TCP/IP 参考模型对比

OSI 是 ISO 提出的七层理论参考框架，TCP/IP 是互联网实际采用的四层协议模型；两者都使用分层思想，但 TCP/IP 将 OSI 的顶部三层合并为应用层、底部两层合并为网络接口层，中间两层一一对应。

## 核心要点

### 分层对应关系

| OSI 层 | TCP/IP 层 | 对应方式 |
|---------|-----------|----------|
| 第 7 层 应用层 | 应用层 | 三层合一 |
| 第 6 层 表示层 | (不存在) | 并入应用层 |
| 第 5 层 会话层 | (不存在) | 并入应用层 |
| 第 4 层 传输层 | 传输层 | 一一对应 |
| 第 3 层 网络层 | 网际层 | 一一对应 |
| 第 2 层 数据链路层 | 网络接口层 | 两层合并 |
| 第 1 层 物理层 | 网络接口层 | 两层合并 |

### 五大核心差异

1. **层数**：OSI 7 层 vs TCP/IP 4 层
2. **设计方法**：OSI 理论驱动（先模型后协议），TCP/IP 实践驱动（先协议后模型）
3. **会话层/表示层**：OSI 独立存在，TCP/IP 不存在（并入应用层）
4. **底层划分**：OSI 分开数据链路和物理层，TCP/IP 合并为网络接口层
5. **标准化**：OSI 是 ISO 标准但未被采用，TCP/IP 是事实标准（IETF RFC）

### 共同设计原则

- 分层架构：每层对上层提供服务，对下层透明
- 传输层和网络层功能高度一致
- 封装/解封装机制
- 协议栈组织方式

## 易错点

- 不要把 TCP/IP 的"网际层"等同于 OSI 的"网络层"的完整功能——OSI 网络层同时支持面向连接和无连接，而 IP 只提供无连接服务
- "TCP/IP 没有表示层和会话层"不等于"这些功能不存在"——它们由应用层协议或库来实现
- OSI 模型虽然未被实际部署，但其分层术语（如"第 2 层交换机""第 3 层路由"）在工程实践中被广泛使用
